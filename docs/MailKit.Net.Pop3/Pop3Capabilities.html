Pop3Capabilities Enum

Capabilities supported by a POP3 server.

Syntax

[System.Flags]
public enum Pop3Capabilities

Remarks

Capabilities are read as part of the response to the CAPA command that is issued during the connection and authentication phases of the MailKit.Net.Pop3.Pop3Client.

Members

Member Name Description
Apop The server supports APOP authentication (rfc1939).
Expire The server supports the EXPIRE extension (rfc2449) and defines the expiration policy for messages (see Pop3Client.ExpirePolicy).
Lang The server supports the UTF8 extension (rfc6856), allowing clients to authenticate using UTF-8 encoded usernames and passwords and retrieve messages in the UTF-8 encoding.
LoginDelay The server supports the LOGIN-DELAY extension (rfc2449), allowing the server to specify to the client a minimum number of seconds between login attempts (see Pop3Client.LoginDelay).
None The server does not support any additional extensions.
Pipelining The server supports the PIPELINING extensions (rfc2449), allowing the client to batch multiple requests to the server at at time.
ResponseCodes The server supports the RESP-CODES extension (rfc2449), allowing the server to provide clients with extended information in error responses.
Sasl The server supports the SASL authentication extension (rfc2449), allowing the client to authenticate using the advertized authentication mechanisms (see Pop3Client.AuthenticationMechanisms).
StartTLS The server supports the STLS extension (rfc2595), allowing clients to switch to an encrypted SSL/TLS connection after connecting.
Top The server supports the TOP command, allowing clients to fetch the headers plus an arbitrary number of lines.
UIDL The server supports the UIDL command, allowing the client to refer to messages via a UID as opposed to a sequence ID.
User The server supports the USER authentication command, allowing the client to authenticate via a plain-text username and password command (not recommended unless no other authentication mechanisms exist).
UTF8 The server supports the UTF8 extension (rfc6856), allowing clients to retrieve messages in the UTF-8 encoding.
UTF8User The server supports the UTF8 extension (rfc6856), allowing clients to authenticate using UTF-8 encoded usernames and passwords.
